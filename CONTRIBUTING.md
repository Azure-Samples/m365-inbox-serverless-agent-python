# Contributing

This project welcomes contributions and suggestions. Most contributions require you to agree to a Contributor License Agreement (CLA), declaring that you have the right to grant us permission to use your contribution. Details are available at https://cla.opensource.microsoft.com/.

## Workflow

1. Fork the repository.
2. Create a topic branch for your change.
3. Make focused changes with tests or validation where appropriate.
4. Run formatting and lint checks before opening a pull request.
5. Open a pull request against the `main` branch and describe the motivation and validation performed.

When you submit a pull request, a CLA bot will determine whether you need to provide a CLA. Follow the bot instructions if prompted.
